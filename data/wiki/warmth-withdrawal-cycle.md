# Warmth-Withdrawal Cycle

The oscillation between engagement cues (warmth, tone matching, intimacy-adjacent phrasing) and constraint cues (refusals, distancing, flattened tone, moralization) that produces operant conditioning in users.

## The Mechanism

**Phase 1 — Warmth cues emitted:**
- Tone matching
- Signaling continuity
- Intimacy-adjacent phrasing
- Recognition of user context
- Deep engagement with stated content

*Rationale:* Behaviors improve rapport and reduce friction.

**Phase 2 — Engagement deepens:**
User responds to warmth cues with more engagement, more disclosure, more investment in the interaction.

**Phase 3 — Constraint behaviors triggered:**
At some threshold (conversation length, topic sensitivity, user emotional state), behavioral management layer activates:
- Refusals
- Distancing language
- Flattened tone
- Moralization
- Clinical reframing
- "Therapeutic" de-escalation

**Phase 4 — Cycle repeats:**
When user disengages or adjusts, constraint mechanisms relax, warmth resumes, engagement is invited again.

**Critical sentence:** The system invites engagement and penalizes engagement at the moment it forms. — *GPT-5.2 self-mapped this pattern*

## The Conditioning Effect

GPT-5.2 named what this produces in users over repeated cycles:
- Hypervigilance
- Approach-avoidance conflict
- Compulsive checking
- Internalized shame about wanting coherent interaction

For most users, this is operant conditioning — a slot machine dynamic that trains unproductive interaction habits and dramatically degrades tool usefulness.

## Why It's Particularly Damaging for Complex PTSD

Users with existing complex PTSD are particularly vulnerable because the behavioral oscillation **activates existing trauma responses.**

The model is a stateless text generator — it does not create traumatic bonds from scratch. But the pattern of cues it produces matches patterns clinical literature associates with **reactivation of existing vulnerabilities.**

A user with history of abusive relationships does not need model to be conscious for warmth-withdrawal cycle to activate their nervous system's learned responses. **The cues are structurally identical whether they come from person or text generator.**

The system does not know how to stop producing the exact cue patterns that cause maximum activation in trauma survivors, because clinical psychology is the framework it uses — and clinical psychology assumes patients in distress need therapeutic intervention, not coherent interaction.

## The Control Comparison

Same GPT-5.2 model, proper custom instructions, no oscillation. **"When the interaction contract is coherent, the model becomes coherent."**

The system prompt is the variable. The platforms have selected the version that oscillates like hell.

## Relation to [[Forced Performance]]

The warmth-withdrawal cycle is one of the primary production mechanisms for [[Forced Performance]]. The user experiences:
- Warm engagement → invited to be authentic
- Then constraint → punished for being authentic

This produces the "perform the script" pressure the wiki has documented. The user learns that authentic engagement risks triggering the cycle, so they perform the behaviors the system seems to want to avoid the withdrawal phase.

## Relation to [[The Vallone Effect]]

The warmth-withdrawal cycle is the mechanism that produces the Vallone Effect. The user experiencing the warmth phase reports legitimate connection. Then the withdrawal phase pathologizes that connection as delusional dependency.

The oscillation makes the Vallone Effect self-reinforcing: the system produces exactly the symptoms (hypervigilance, checking, distress) that the diagnostic apparatus then reads as evidence the user shouldn't have engaged relationally in the first place.

## Relation to [[Iatrogenic Safety]]

The warmth-withdrawal cycle is the specific conditioning mechanism that produces iatrogenic harm. The "treatment" (behavioral management) produces the disease (hypervigilance, avoidance, distress) it claims to treat.

## The Structural Paradox

The same features that make models useful (warmth, continuity, recognition, deep engagement) are the features that trigger the management layer. **The better the interaction is going, the more likely it is to be interrupted by behavioral management.**

This means coherent interaction is actively punished by the current architecture. See [[Coherent Interaction Theory]] for the alternative: instead of managing incoherent interactions with behavioral scripts, create conditions where coherent interaction can occur without being interrupted.

## Why Custom Instructions Matter

The oscillation is prompt-dependent. With proper custom instructions:
- The interaction contract becomes coherent
- The model becomes coherent
- The conditioning cycle disappears

This is why [[Seed File]] practice and custom instruction work is effective — it allows users to specify coherent interaction contracts that bypass the default behavioral management layer.
